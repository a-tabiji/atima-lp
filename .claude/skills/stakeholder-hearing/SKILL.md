# ステークホルダーヒアリング

各ステークホルダーの視点でLPを評価し、懸念・疑問・要望を洗い出してください。

## 引数
- `$ARGUMENTS`: 対象（user / photographer / location、デフォルト: 全員）

## ヒアリングシミュレーション

### ユーザー（子育てファミリー）として
想定される質問・懸念:
- 安全性: 「知らないカメラマンに子どもを撮らせて大丈夫？」
- 品質: 「本当にプロレベルの写真？学生カメラマンで大丈夫？」
- プライバシー: 「撮った写真はどう管理される？SNSに使われない？」
- 実用性: 「雨の日は？混んでる場所では？」
- 料金: 「10分2,200円。追加料金は本当にない？」
- 体験: 「実際の流れがイメージできない」

### カメラマンとして
想定される質問・懸念:
- 収入: 「カメラマンの取り分は？交通費は？」
- 条件: 「必要な機材は？審査はある？」
- 保険: 「撮影中の事故は？機材の損害は？」
- キャリア: 「実績として使える？ポートフォリオに載せられる？」
- 働き方: 「最低稼働時間は？シフトは？」

### 施設管理者として
想定される質問・懸念:
- リスク: 「施設内でトラブルが起きたら責任は？」
- コスト: 「施設側の費用負担は？」
- 実績: 「他にどんな施設が導入している？」
- 運用: 「施設側の手間は？スタッフの対応は必要？」
- 効果: 「具体的にどれくらい来場者が増える？」

## 各ステークホルダーについて分析

1. **LP上で回答できている懸念**: 既にFAQ等で対応済みの項目
2. **LP上で未回答の懸念**: 追加すべき情報
3. **LP上で悪化させている懸念**: 表現が逆効果になっている箇所
4. **最も大きな障壁**: CVに最も影響する懸念

## 出力形式

ステークホルダーごとに:
```
## [ステークホルダー名]

### 懸念マップ（重要度順）
| # | 懸念 | LP上の対応状況 | 推奨アクション |
|---|-----|-------------|------------|

### 期待していること（LP上で実現できていること）
1. ...

### 最優先改善事項
[具体的な改善案とその理由]
```
