# UX レビュー

LP全体のユーザー体験を分析し、改善提案をしてください。

## 分析フレームワーク

### 1. ファーストビュー（Above the fold）
- ヒーローセクションで「何のサービスか」「誰向けか」「何ができるか」が3秒で伝わるか
- CTAの位置と文言は適切か
- ビジュアルが価値提案を補強しているか

### 2. 情報設計・導線
- セクション順序はユーザーの思考プロセス（認知→興味→理解→信頼→行動）に沿っているか
- 各セクション間の論理的つながり
- スクロール誘導の仕組み

### 3. CTA設計
- CTAボタンの数、配置、間隔は適切か
- ボタン文言が行動を促しているか（「詳しく見る」vs「モニターに応募する」）
- CTA到達までのステップ数

### 4. 認知負荷
- テキスト量は適切か（読まれる量か）
- 視覚的な休息ポイントはあるか
- 選択肢の数（ヒックの法則）

### 5. 信頼構築
- 社会的証明（ユーザーの声）の説得力
- 料金の透明性
- 不安を解消する要素（FAQ、安心感）

### 6. モバイルUX
- タップ操作の快適さ
- スクロール量は適切か
- 重要情報の視認性

## 出力形式

### 評価サマリー
各項目を5段階で評価し、総合スコアを算出。

### 改善提案
優先度（高/中/低）と期待効果付きで、具体的な改善案を列挙。
コード変更が伴う場合は実装イメージも提示。

### クイックウィン
すぐに実装でき、効果が見込める改善を3つ厳選。
