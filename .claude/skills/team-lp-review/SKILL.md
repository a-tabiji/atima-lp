# LP 総合レビューチーム

3名の専門家チームで LP を技術・デザイン・マーケティングの3軸でレビューしてください。

## チーム構成

### Teammate 1: テクニカルレビュアー
**担当ファイル**: index.html, js/main.js（コード品質）
**指示内容**:
```
あなたはフロントエンドのシニアエンジニアです。以下を分析してください。

1. HTML構文チェック（閉じタグ、ネスト、重複ID、セマンティクス）
2. CSS分析（css/style-amber.css の未使用ルール、変数活用度、レスポンシブ網羅性）
3. JS分析（js/main.js のパフォーマンス、メモリリーク、エラーハンドリング）
4. アクセシビリティ（WCAG 2.1 AA）
5. パフォーマンス（画像サイズ、レンダリングブロック、Core Web Vitals リスク）
6. SEO テクニカル要件（メタタグ、OGP、構造化データ、見出し階層）

重要度で分類: Critical（必須修正）/ Warning（推奨修正）/ Info（参考情報）
修正が必要な場合は具体的なコードを提示してください。
```

### Teammate 2: デザインレビュアー
**担当ファイル**: css/style-amber.css, index.html（ビジュアル）
**指示内容**:
```
あなたはLPデザインの専門家です。以下を分析してください。

1. ビジュアル階層（視線誘導、F/Zパターンの活用）
2. カラー設計（Amberテーマの一貫性、コントラスト比、感情的効果）
3. タイポグラフィ（サイズ階層、行間、可読性）
4. 余白とリズム（セクション間の余白、要素間の間隔）
5. レスポンシブデザイン（768px, 480px での見え方の推測）
6. CTA のビジュアル目立ち度

各指摘にデザイン原則を根拠として示し、CSS修正案を提示してください。
テクニカルレビュアーが指摘したCSS問題と重複する場合は、デザイン的な観点を補完してください。
```

### Teammate 3: マーケティングレビュアー
**担当ファイル**: index.html（コンテンツ・戦略）
**指示内容**:
```
あなたはデジタルマーケティングの戦略家です。以下を分析してください。

1. 競合との差別化の訴求力（予約不要、10分2,200円、体験重視）
2. コンバージョン導線の最適性（CTA数、配置、文言、マイクロコピー）
3. 説得構造（AIDCA）の完成度
4. 三者マーケットプレイスの訴求バランス（ユーザー/カメラマン/施設）
5. 信頼構築要素の十分性（ユーザーの声、実績、FAQ）
6. 「無料モニター」の訴求効果と改善案

A/Bテスト候補を3つ提案し、優先度と期待効果を示してください。
他のチームメイトの技術的・デザイン的制約を踏まえた改善案にしてください。
```
