# ペルソナインタビュー

指定されたペルソナになりきり、LPを訪問した体験をシミュレーションしてください。

## 引数
- `$ARGUMENTS`: ペルソナ名（省略時は全ペルソナを順番に実施）

## ペルソナ定義

### ペルソナA: 子育てママ（メインターゲット）
- 32歳、2歳の子どもあり、共働き
- 週末は公園や商業施設で過ごすことが多い
- スマホ中心の情報収集、SNS（Instagram）をよく利用
- 写真は好きだが、撮影に時間を割く余裕がない
- 出張撮影サービスは聞いたことがあるが、高額・予約面倒のイメージ

### ペルソナB: 写真専門学校生（カメラマン候補）
- 22歳、写真専門学校2年生
- ポートフォリオを充実させたい
- 副業・バイトで収入を得たい
- 人物撮影の実践経験が少ない
- 新しいサービスへの好奇心が強い

### ペルソナC: 公園管理者（施設提供者）
- 45歳、市立公園の運営担当
- 来場者数の増加がKPI
- 新規企画の導入には稟議が必要
- リスク回避的、実績を重視
- デジタルリテラシーは中程度

## シミュレーション手順

各ペルソナについて、LPを上から下まで読み進めながら:

1. **第一印象**（ヒーローセクション）
   - 何のサービスか理解できたか？
   - 自分向けだと感じたか？

2. **セクションごとの反応**
   - 各セクションで「なるほど」「よくわからない」「不安」「興味が湧いた」等の感情を記録
   - 離脱しそうになるポイントを特定

3. **CTA到達時の心理状態**
   - 応募/問い合わせする気持ちになったか？
   - 残っている疑問・不安は何か？

4. **総合評価**
   - このサービスを使いたい/参加したいと思ったか？（10段階）
   - 決め手になった要素 / 障壁になった要素

## 出力形式

ペルソナごとに以下を出力:
```
## ペルソナ[X]: [名前]

### スクロール体験シミュレーション
[セクション名] → [感情・反応] → [思考]
...

### 離脱リスクポイント
1. [具体的なポイントと理由]

### 改善提案
1. [ペルソナ固有の改善案]
```
